<p>위 그림은 크기가 5인 정수 삼각형의 한 모습이다.</p>

<p>맨 위층 7부터 시작해서 아래에 있는 수 중 하나를 선택하여 아래층으로 내려올 때, 이제까지 선택된 수의 합이 최대가 되는 경로를 구하는 프로그램을 작성하라. 아래층에 있는 수는 현재 층에서 선택된 수의 대각선 왼쪽 또는 대각선 오른쪽에 있는 것 중에서만 선택할 수 있다.</p>

<p>삼각형의 크기는 1 이상 500 이하이다. 삼각형을 이루고 있는 각 수는 모두 정수이며, 범위는 0 이상 9999 이하이다.</p>

### 입력 

 <p>첫째 줄에 삼각형의 크기 n(1 ≤ n ≤ 500)이 주어지고, 둘째 줄부터 n+1번째 줄까지 정수 삼각형이 주어진다.</p>

### 출력 

 <p>첫째 줄에 합이 최대가 되는 경로에 있는 수의 합을 출력한다.</p>

## 문제 해결 아이디어

삼각형 위에서부터 찾지 말고, 아래에서부터 최대를 고르는 식으로 찾아보려고 한다.   
입력으로 들어오는 값들을 tri에 2차원 리스트로 저장한다.   
그리고 i-1층의 값들을 i층의 값들 중 큰 걸로 교체하면서 0층까지 교체한다.    

즉, 아래 그림으로 예시를 들어보면   
[3, 0] 위치의 값은 4와 5 중에 자신과 더해 더 커지는 값으로 교체한다. ⇒ 7   
[3, 1] 위치의 값은 5와 2 중에 자신과 더해 더 커지는 값으로 교체한다. ⇒ 12   
[2, 0] 위치의 값은 7과 12 중에 자신과 더해 더 커지는 값으로 교체한다. ⇒ 20

이런 과정을 거치게 되면 [0, 0] 위치(가장 꼭대기)엔 가장 큰 값이 남게 된다.    

## Input 반례 (해결 과정)

원래는 `tri` 값을 교체하기 전에 `if i+1 < n and j+1 < len(tri[i+1]):` 이렇게 조건을 검색했었는데, 생각해보니까 어차피 가장 아래 층의 위 층에서부터 반복을 시작하고, 삼각형은 아래 층이 개수가 하나씩 더 많기 때문에 불필요한 조건문이었다.

## 최종 코드

```python
import sys
n = int(sys.stdin.readline())
tri = []
for _ in range(n):
    tri.append(list(map(int, sys.stdin.readline().split())))

for i in range(n-2, -1, -1):
    for j in range(len(tri[i])):
        tri[i][j] = max(tri[i][j]+tri[i+1][j],tri[i][j]+tri[i+1][j+1])
print(tri[0][0])
```
